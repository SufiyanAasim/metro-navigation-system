# Metro Navigation System — v1.1.5

**Codename: Teal**

## 📖 Overview

Teal is the current release of the Metro Navigation System — a UI-refinement milestone that replaces standard Windows dialogs with a custom dark theme, introduces a dedicated printable receipt flow, and restructures the options menu for a cleaner wizard experience.

## 🎯 Objectives

- Replace standard Windows message boxes with a themed custom dialog.
- Build a dedicated, printable receipt flow.
- Replace dropdown combo boxes with custom graphical destination-select buttons.
- Restructure the options menu order and simplify the Stations page.

## ✨ Added

### 🎨 Premium Custom Message Boxes
- Replaced standard gray Windows OS message boxes with a custom styled dialog (`CustomMessageBox.cs`).
- Renders a deep dark blue background (`#0F192D`), Segoe UI typography, and a flat Teal OK button (`#0D9488`).

### 🧭 Interactive & Clipless Options Screen
- **Shifted Option Layout** — moved option buttons left (X=120) and description labels left (X=340) to prevent description text overflow or clipping.
- **Clickable Descriptions** — description labels are now fully interactive, showing a Hand cursor and navigating directly to the respective screen when clicked.

### ⚖️ Equidistant Control Layouts
- Repositioned the Yes/No buttons on the Receipt screen to be perfectly equidistant from each other and the screen borders (Yes at X=180, No at X=478).

### 🧾 Custom Destination Select Buttons
- Replaced dropdown combo boxes on the Shortest Path screen with custom Sky Blue (`btn_first_destination.png`) and Amber (`btn_final_destination.png`) destination buttons that spawn a custom dark dropdown menu of stations, displaying choices on bold white text overlays on the map.

### 🖨️ Dedicated Printable Receipt Screen Flow
- Integrated a custom `btn_print.png` Print button matching the app's visual style.
- Clicking **Yes** to generate a receipt hides the Yes/No buttons and query label, showing the printed ticket details with only the Print button visible.
- Clicking **No** immediately routes back to the main options screen.

### 🚉 Stations Page Map Pin Clicks & Dropdown Cleanup
- Removed the old station dropdown combobox (`comboBox1`) and its keyboard event handlers — the Stations page now relies solely on the **Show Stations on Map** toggle.
- Clicking any station pin on the map now displays a custom dialog with the station's full name.

## 🔧 Changed

- **Restructured Menu Layout & Wizard Loop** — re-positioned options so "Metro Map" appears first and "Stations" second; corrected the wizard flow to `Welcome → Choices → Metro Map → Stations → Shortest Route → Receipt`.
- **Software Engineer Title Renaming** — changed the developer credits role title from "Lead Developer" to "Software Engineer" on both the Credits screen and in the README.

## 🏛️ Architecture Decisions

**Custom UI components over stock controls.** `CustomMessageBox` establishes the pattern of replacing stock WinForms controls with themed equivalents where the standard look breaks visual consistency — the same reasoning behind the custom destination-select buttons introduced in this release.

## ⚠️ Known Limitations

- No automated test suite (see [tests/README.md](../../tests/README.md) for how to add one).
- No relational database — trip history is a flat text-file log (see [database.md](../database.md)).
- No accounts or authentication — single-user, offline only (see [authentication.md](../authentication.md)).
- Windows-only (WinForms / .NET Framework).
